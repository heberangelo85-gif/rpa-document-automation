# 🤖 RPA Document Processing & Archiving Automation

## 📌 Overview

This project demonstrates the implementation of an RPA (Robotic Process Automation) solution designed to automate the ingestion and archiving of business-critical documents into an ERP system.

The solution was developed in a financial environment, where document processing is a high-volume and time-sensitive operation.

---

## 🎯 Business Problem

Before automation:

* Document processing was fully manual
* Required at least **2 employees**
* Time-consuming (~3 minutes per document)
* Limited to business hours
* Prone to human error and delays

---

## ⚙️ Solution

An RPA solution was implemented using **Microsoft Power Automate**, based on structured business requirements.

### Key Features:

* Automated document ingestion into ERP
* Classification of document types (e.g., contracts, registration forms, credit documents)
* Continuous processing (24/7)
* Error handling and monitoring
* Integration with existing business workflows

---

## 📊 Impact & Results

### 📈 Operational Metrics

* **4,800+ documents processed per month**
* **14,000+ documents processed (total)**
* **97% success rate**

---

### ⏱️ Efficiency Gains

* Average manual processing time: **3 min 39 sec per document**
* Total time saved:

👉 **~290 hours/month**
👉 Equivalent to **~1.5 full-time employees (FTE)**

---

### 💰 Business Impact

* Reduced dependency on manual labor
* Enabled **24/7 automated processing**
* Increased scalability and operational reliability

## 🏗️ Process Flow (Simplified)

1. Document generated by business process
2. RPA identifies document type
3. Data is validated and structured
4. Document is uploaded and archived in ERP
5. Status is logged and monitored
